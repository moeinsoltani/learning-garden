# Phase 10: libvirt — The Management Layer

Raw QEMU command lines are powerful but unwieldy and ephemeral. **libvirt** is the
management layer that turns a VM into a persistent, named object with a stable API,
generating the QEMU command line for you. This phase covers libvirt's architecture and
`virsh`, the domain XML (the canonical VM definition), storage pools/volumes, virtual
networks (which tie back to the networking curriculum), and `virt-install` plus the
desktop tools.
