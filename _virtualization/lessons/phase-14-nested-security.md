# Phase 14: Nested Virtualization & Security

Two related themes: running KVM *inside* KVM (essential for labs, CI, and this very
curriculum), and locking down the large, sensitive QEMU process. This phase covers nested
virtualization, sVirt (confining QEMU with mandatory access control), QEMU hardening and
sandboxing, and confidential computing (encrypting the guest *from* the host).
